LifeLens Mobile App - Product & Technical Requirements

1. Product Requirements Document (PRD)

1.1 Product Vision

To become India's most accessible, AI-first lifestyle coach that empowers users to prevent chronic diseases through simple, daily tracking of holistic health metrics (Screen time, Sleep, Diet, Environment, Activity). "Prevention is better than cure."

1.2 Target Audience

  • Primary: Indian users aged 18-45 in Tier 1 & 2 cities.
  • Language: Hindi & English speakers.
  • Tech Literacy: Mixed. Ranging from tech-savvy professionals to users with basic smartphone knowledge.
  • Key Characteristic: Concerned about health but overwhelmed by complex medical jargon or expensive tests.

1.3 MVP Goals

  1. Acquisition: Achieve 10,000 downloads in Delhi NCR region.
  2. Engagement: Daily Active Users (DAU) tracking at least 3 metrics.
  3. Retention: Users completing the "Health Assessment" and viewing their risk report.

1.4 Key Features (MVP)

  1. Bilingual Onboarding: Seamless language selection (Hindi/English) and phone number login (OTP).
  2. "Check Your LifeLens" Assessment: The core hook. A visual, chat-like wizard to assess initial health risks.
  3. Visual Dashboard: A card-based, icon-heavy home screen showing daily status of 7 key metrics.
  4. One-Tap Tracking: Simplified input methods (sliders, +/- buttons) for logging water, sleep, etc.
  5. Real-time Environment: Auto-fetch AQI and Weather based on location.
  6. AI Health Report: Simple, color-coded risk cards (Green/Orange/Red) with actionable "Desi Nudges" (e.g., "Drink water now," "Take a walk").

2. Software Requirements Specification (SRS)

2.1 Functional Requirements

2.1.1 Authentication & Onboarding

  • FR-01: System MUST allow users to sign up/login using Phone Number + OTP (Firebase Auth).
  • FR-02: System MUST prompt for Language Selection (Hindi/English) immediately upon first launch.
  • FR-03: System MUST store user profile locally (Name, Age, Gender, Location) and sync to backend when online.

2.1.2 Health Assessment (The Hook)

  • FR-04: App MUST present a "Check Your LifeLens" wizard for first-time users.
  • FR-05: Wizard MUST collect: BMI data (Height/Weight), Occupation, Activity Level, Diet Type.
  • FR-06: System MUST calculate an initial "Risk Score" (0-100) locally on the device immediately after assessment.

2.1.3 Dashboard & Tracking

  • FR-07: Dashboard MUST display 7 metric cards: Screen Time, Sleep, Water, Steps, Exercise, Diet, AQI.
  • FR-08: Users MUST be able to update metrics with minimal typing (use sliders, steppers, or quick-select icons).
  • FR-09: App MUST auto-fetch AQI using OpenWeatherMap API based on GPS permission.
  • FR-10: App MUST support "Offline Mode" - allowing users to track data without internet, syncing when connection is restored.

2.1.4 AI Insights & Reports

  • FR-11: System MUST generate a daily "Health Score" based on tracked metrics.
  • FR-12: System MUST display "Nudges" (Notifications/In-app cards) triggered by logic (e.g., if Water < 2L -> "Hydrate now").
  • FR-13: Reports MUST use traffic light colors (Green=Good, Yellow=Caution, Red=Danger) for universal understanding.

2.2 Non-Functional Requirements (NFR)

  • NFR-01 (Usability): UI elements MUST be large and touch-friendly (min 44x44pt targets). Text MUST be concise. Icons MUST be used alongside text for all major actions (for low-literacy support).
  • NFR-02 (Performance): App launch time MUST be under 2 seconds.
  • NFR-03 (Localization): All static text MUST be stored in localization files (en.json, hi.json) for instant toggling.
  • NFR-04 (Data Privacy): Health data MUST be stored encrypted in local storage (AsyncStorage/MMKV).
  • NFR-05 (Connectivity): App MUST function 100% for tracking features even in "Airplane Mode".

2.3 Backend & API Requirements (High Level)

  • Database: NoSQL (MongoDB/Firestore) for flexible user profile and daily logs.
  • API Layer: REST or GraphQL endpoints for:
    • POST /user/profile (Sync profile)
    • POST /user/metrics (Sync daily logs)
    • GET /content/nudges (Fetch dynamic health tips)
  • Auth: JWT based authentication linked to Phone Number.

3. User Stories (For Agile Development)

Epic 1: Onboarding & First Impressions

  • US-1.1: As a Hindi-speaking user, I want to select my language on the first screen, so that I can understand the rest of the app setup.
  • US-1.2: As a user, I want to login via OTP, so that I don't have to remember complex passwords.
  • US-1.3: As a new user, I want to complete a quick health quiz, so that the app knows my baseline health risks immediately.

Epic 2: Daily Tracking (The Core Loop)

  • US-2.1: As a busy professional, I want to log my water intake with one tap, so that I don't waste time typing.
  • US-2.2: As a user in Delhi, I want to see the current AQI automatically, so I know if I should wear a mask.
  • US-2.3: As a user, I want to see a red warning if my screen time is too high, so I am reminded to take a break.

Epic 3: Insights & Retention

  • US-3.1: As a user, I want to see a simple "Health Score" (e.g., 85/100), so I know how well I did today at a glance.
  • US-3.2: As a user, I want to receive a notification to drink water, but only if I haven't logged enough water by 2 PM.

4. UI/UX Guidelines for "Education-Agnostic" Design

  1. Voice & Tone: Empathetic, encouraging, and simple. Avoid medical jargon (Use "Eye Strain" instead of "Computer Vision Syndrome").
  2. Visuals: Use illustrations for every category (e.g., a Glass icon for water, a Bed icon for sleep).
  3. Color Semantics: Strictly adhere to Green/Orange/Red for status. Do not use these colors for decoration to avoid confusion.
  4. Navigation: Bottom Tab Bar with max 4 tabs: Home (Tracker), Report (Analysis), Learn (Tips), Profile.
